# Security Policy

## Supported versions

Security-related corrections are applied to the latest released version of the guidelines and repository tooling.

## Reporting a security issue

Do not place credentials, private environment details, customer data, or exploitable workflow information in a public issue.

Use GitHub private vulnerability reporting when it is available. Otherwise, open a minimal public issue requesting a private maintainer contact without including sensitive details.

Relevant reports include:

- secrets or private data accidentally published in documentation or examples;
- a validation or workflow weakness that could execute untrusted content;
- guidance that would systematically encourage an unsafe permission or data-handling model;
- a repository configuration issue with meaningful security impact.

Ordinary wording errors and non-sensitive rule proposals should use the normal issue templates.
