# Frontend Guidelines

Before frontend development, read:

- `docs/frontend-guidelines/AGENTS.md`
- `docs/frontend-guidelines/CORE_RULES.md`
- `docs/frontend-guidelines/FRONTEND_SPEC.md` sections and detailed chapters selected by the task map
- `docs/frontend-guidelines/docs/10-validation-checklist.md` before delivery
- the project root `PROJECT_FRONTEND_PROFILE.md`

Precedence: non-waivable legal, security, privacy, and accessibility requirements > current user requirements > documented project constraints > established reasonable patterns > shared guidelines > framework or UI-library defaults.

For material changes, document the root cause, fix layer, affected sibling surfaces, applicable rule IDs, state matrix, and validation plan. Record any justified deviation in `PROJECT_FRONTEND_PROFILE.md`.
