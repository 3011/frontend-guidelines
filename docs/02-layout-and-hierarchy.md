# 布局与视觉层级

## 间距系统

项目可以使用自己的尺寸体系，但必须保留可辨识的语义层级：

- 紧密关联元素：小间距；
- 同组字段或控件：标准间距；
- 独立语义区块：更大间距；
- 页面主要区域：最大间距。

推荐参考值为 8 / 16 / 24 / 32 像素，但规则关注比例与一致性，不绑定具体实现。

## 分隔线

分隔线只辅助层级，不能替代间距和标题。它的两侧必须有足够留白。

错误：

```text
说明文字
----------------
下一节标题
```

正确：

```text
说明文字

----------------

下一节标题
```

## 对齐与稳定性

关键操作、状态、列标识和固定含义的信息应保持稳定位置。以下变化不得导致无意义跳动：

- 文本从短变长；
- 状态图标出现或消失；
- 数据刷新；
- 展开和收起；
- 加载占位切换为真实内容。

## 长内容

必须主动验证：

- 中文长词组；
- 没有空格的标识符；
- 较长数字或地址；
- 浏览器放大；
- 窄屏；
- 翻译后长度增加。

可以换行、截断、滚动或重排，但关键操作不可消失。截断内容需要可访问的完整查看方式。

## 密度

高密度不是把所有空白删除。密度应由任务频率和扫描需求决定：

- 高频运维列表可更紧凑；
- 创建、编辑和风险确认需要更清晰的分组；
- 首屏不应同时竞争多个主要操作。
