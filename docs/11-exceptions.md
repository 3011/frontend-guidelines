# 规范偏离与例外

规范不是替代产品判断。项目可以偏离，但必须显式记录。

## 可接受的偏离

常见合理原因：

- 行业或法律强制要求；
- 目标设备能力限制；
- 无障碍需求要求不同交互；
- 项目已经存在且无法立即迁移的稳定模式；
- 用户在当前任务中明确要求不同方案。

“实现更快”“组件默认如此”“只在这里特殊”不是充分理由。

## 记录格式

```text
偏离规则：FG-...
范围：页面、流程或组件使用位置
原因：
用户影响：
替代保护措施：
验证方式：
复查条件或到期时间：
```

## 临时偏离

临时偏离必须有复查条件。条件满足后应删除偏离，而不是让临时方案永久化。

## 规则本身不适用

若某条规则与任务无关，可标记“不适用”，无需创建偏离记录。但不得把“未验证”写成“不适用”。
