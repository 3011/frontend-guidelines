# 用户界面文案

## 用户语言

界面只展示用户完成任务所需的信息。

禁止直接展示：

- “权限由后端强制校验”；
- “此功能后续开发”；
- “兼容旧接口”；
- “这里使用某组件实现”；
- 技术债、版本迁移和开发备注。

这些内容属于开发文档、版本说明或运维信息，不属于普通业务页面。

## 操作名称

优先使用“动词 + 对象”：

- 保存设置；
- 创建任务；
- 停止扫描；
- 删除记录；
- 重新连接。

只有上下文极其明确时才使用“确定”“完成”等短词。

## 说明文字

说明文字应帮助用户做决定，回答：

- 这个选项改变什么；
- 何时需要修改；
- 默认行为是什么；
- 是否存在风险。

不要重复标签，不要解释实现。

## 错误文案

用户文案应先说明可理解的问题，再提供恢复方法。内部标识可作为次要诊断信息，但不能替代说明。

## 一致术语

同一概念在导航、标题、按钮和状态中使用同一名称。术语变更应全局检查，不应在单个页面创建别名。
